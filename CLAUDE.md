# AeroSpace

A fork of [AeroSpace](https://github.com/nikitabobko/AeroSpace), an i3-like tiling window manager for macOS.

## Prerequisites

- **swiftly** (Swift toolchain manager): `brew install swiftly && swiftly init --skip-install --assume-yes && swiftly install`
  - The project pins Swift version in `.swift-version` (currently 6.2.3)
  - All `swift` commands in build scripts run through `swiftly run swift` automatically (see `script/setup.sh`)

## Build (debug)

```bash
./build-debug.sh
```

This runs `swift build` (via swiftly), builds the test target, and copies binaries to `.debug/`.

To build with warnings as errors: `./build-debug.sh -Xswiftc -warnings-as-errors`

## Run (debug)

```bash
./run-debug.sh
```

Builds debug and launches `AeroSpaceApp`.

## Tests

Run unit tests only (fast, use this during development):

```bash
./run-swift-test.sh
```

Run the full test suite (builds, tests, checks formatting, checks codegen is up to date):

```bash
./run-tests.sh
```

The full suite runs: `build-debug.sh` -> `run-swift-test.sh` -> CLI smoke tests -> `format.sh --check-uncommitted-files` -> `generate.sh` -> `check-uncommitted-files.sh`

## Build and install to /Applications

1. Regenerate the Xcode project and generated sources:

```bash
./generate.sh --ignore-cmd-help --ignore-shell-parser
```

2. Build with xcodebuild using the swiftly-installed toolchain (Xcode's bundled compiler is too old for some dependencies). Use ad-hoc signing (`CODE_SIGN_IDENTITY="-"`):

```bash
toolchain="$(plutil -extract CFBundleIdentifier raw ~/Library/Developer/Toolchains/swift-6.2.3-RELEASE.xctoolchain/Info.plist)"
xcodebuild -toolchain "$toolchain" clean build \
    -scheme AeroSpace \
    -destination "generic/platform=macOS" \
    -configuration Release \
    -derivedDataPath .xcode-build \
    CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=YES
```

3. Kill the running AeroSpace, replace the app, and relaunch:

```bash
killall AeroSpace 2>/dev/null
cp -rf .xcode-build/Build/Products/Release/AeroSpace.app /Applications/
open /Applications/AeroSpace.app
```

## Build (release)

```bash
./build-release.sh --codesign-identity -
```

Use `--codesign-identity -` to sign for local use only (no certificate). This uses `xcodebuild` for the .app bundle (universal binary arm64+x86_64) and `swift build -c release` for the CLI.

The `.xcodeproj` is generated by XcodeGen from `project.yml` (run `./generate.sh` to regenerate).

## Code generation

```bash
./generate.sh
```

Generates: version info, subcommand descriptions, git hash, shell parser (ANTLR), and the Xcode project.

Use `./generate.sh --ignore-xcodeproj --ignore-cmd-help --ignore-shell-parser` for a fast partial regen (this is what `build-debug.sh` does).

## Formatting

```bash
./format.sh
```

Runs SwiftFormat and SwiftLint (auto-downloads both via `script/install-dep.sh`).

## Project structure

- `Sources/AppBundle/` - Main app logic (window management, layouts, commands)
- `Sources/AeroSpaceApp/` - App entry point
- `Sources/Cli/` - CLI (`aerospace` command)
- `Sources/Common/` - Shared utilities between app and CLI
- `Sources/PrivateApi/` - C headers for macOS private APIs (e.g., `_AXUIElementGetWindow`)
- `Sources/AppBundleTests/` - Unit tests
- `Package.swift` - SPM package definition (Swift 6.2, macOS 13+ deployment target)
- `project.yml` - XcodeGen spec for generating `AeroSpace.xcodeproj`

## Development workflow

This project follows **test-driven development**. Every feature or behavior change must include corresponding tests. Write tests first or alongside the implementation — never ship code without tests.

## Key architectural notes

- The `script/setup.sh` file is sourced by all scripts. It nukes `$PATH` to only `/bin:/usr/bin` plus `.deps/bin`, ensuring reproducible builds. The `swift` shell function routes through `swiftly`.
- Debug builds use SPM directly (`swift build`). Release builds use `xcodebuild` for the .app and `swift build -c release` for the CLI.
- Tests are SPM-based: `swift test` runs `AppBundleTests`.
